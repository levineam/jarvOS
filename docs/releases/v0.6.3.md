# jarvOS v0.6.3

## Summary

v0.6.3 is a patch release for jarvOS's GBrain-first provider lane and the
reconciled public secondbrain surfaces around it. Install it if you want a
portable, local-first operating layer where structured recall is explicit,
private data stays outside the public repository, and coding agents consume
reviewed context rather than raw personal files.

## What's Included

- Adds a reviewed `@jarvos/ontology` provider pipeline that renders bounded
  hierarchy-of-meaning packets for coding agents. Source-backed secondbrain
  evidence becomes an explicit candidate or inquiry record before promotion.
- Makes the generated LLM-wiki a visible, managed Markdown surface from
  source-backed secondbrain sidecars, with safe rebuild guards and documented
  vault/output defaults.
- Reconciles the public GBrain-first resolver narrative across the front door,
  module inventory, release checks, and integration inventory. `@jarvos/gbrain`
  remains the integration layer for curated import, sync, graph recall,
  retrieval evaluation, and runtime resolver bundles.

## Fixes

- Prefers explicit jarvOS timezone configuration over ambient `TZ` to prevent
  journal and note dates drifting in shells or CI.
- Keeps live ontology data out of publish allowlists and uses public ontology
  examples under `schema/examples/public-ontology/`.
- Isolates release-smoke journal writes from a caller's configured journal
  location.

## Breaking Changes

- None known.

## Known Limitations

- jarvOS is GBrain-first: `@jarvos/gbrain` is a resolver integration and does
  not implement GBrain or ship a private graph. This release is not GBrain
  itself.
- The public repo contains generic code, templates, and fixtures—not private
  vault pages, curated manifests, retrieval-eval questions, or runtime secrets.
- Automatic ingestion of all AI conversations and private vault content remains
  out of scope; imports and ontology promotion are explicit and review-gated.
