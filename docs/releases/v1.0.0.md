# jarvOS v1.0.0

Paperclip release lane: SUP-3548 (clean-machine confidence).

**Status: DRAFT — not published.** `package.json` remains `0.7.0` and there is
no `v1.0.0` git tag. This document exists so the active lane has honest draft
notes while post-v0.7.0 work is tracked under `CHANGELOG.md` `[Unreleased]`.
Andrew approval remains mandatory before any public tag or GitHub Release.

## Summary

v1.0.0 is the clean-machine confidence milestone: make jarvOS installable by a
technically capable stranger, with profile pass/fail criteria, public/private
boundary scans, upgrade/repair guidance, and a clean-machine install canary.
It builds on the shipped v0.7.0 control-plane surface and does not replace
GBrain; jarvOS remains **GBrain-first** via `@jarvos/gbrain` as a **resolver**
integration that **does not implement GBrain** and is **not GBrain itself**.

## What's Included

Landed on `main` since v0.7.0 (still unreleased until this version is cut):

- **#115** — CI actually runs the project's `npm test` suite (with `npm ci`)
  and gates `ci-summary` on that job, so module unit tests and release-trust
  checks block regressions instead of only lint/link/smoke.
- **#116** — Durable landing of the 2026-07-09 strategy and recommendable
  public-release plan artifacts as review drafts, with cold-install and CI
  premises re-checked against shipped v0.7.0.

Still required before a real v1.0.0 cut (not claimed complete here):

- clean-machine install canary with profile-specific pass/fail evidence
- release checklist covering every public install profile
- public/private boundary scan evidence for the cut
- upgrade path / package-name migration story from `jarvos-bootstrap` where needed
- rollback or repair guidance for a stranger install
- documented compatibility matrix for supported runtimes
- `package.json` version bump to `1.0.0`, dated CHANGELOG section, and Andrew
  publication approval

## Fixes

- CI no longer claims a green merge path while skipping the documented
  `npm test` chain (#115).

## Breaking Changes

- None claimed yet. Before the public cut, any package rename or install-path
  change must be called out here explicitly.

## Known Limitations

- This draft does **not** assert clean-machine confidence is done. The lane is
  active; publication is not ready.
- jarvOS remains **GBrain-first**: `@jarvos/gbrain` is a **resolver**
  integration for curated import, sync, graph recall, retrieval evaluation,
  and runtime resolver bundles. It **does not implement GBrain** and is
  **not GBrain itself**.
- Public packages still ship contracts, adapters, and fixtures—not private
  vault content, live Paperclip instance state, or host credentials.
- Public CLI/MCP control-plane surfaces still require a host that binds
  `JARVOS_CONTROL_PLANE_SERVICE_MODULE` and credentials.

## Install / Update

Until v1.0.0 is tagged, install from the latest published release (`v0.7.0`)
or from `main` only if you intentionally want unreleased work:

```bash
git clone https://github.com/levineam/jarvOS.git
cd jarvOS
git checkout v0.7.0   # latest published release
npm test
```

For existing clones on the published line:

```bash
cd jarvOS
git fetch origin
git checkout v0.7.0
npm test
```
