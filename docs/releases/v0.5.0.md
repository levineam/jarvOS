# jarvOS v0.5.0

## Summary

v0.5.0 is the clean public jarvOS release after the accepted fresh-host ship
gate. It consolidates the previously staged CLI/doctor release work with the
v0.5.0 autonomy install-profile lane, so GitHub, the package version, the
changelog, and the public docs all point at one current release.

This release is for people who want the public, local-first jarvOS baseline:
portable Markdown governance, install profiles, doctor checks, skills packs,
agent coding orchestration contracts, journal safety checks, and GBrain-first
context patterns without Andrew's private workspace state.

## What's Included

- Public `jarvos` command router with `jarvos init`, `jarvos doctor`, and
  `jarvos help`, while keeping the older bootstrap aliases available.
- `minimal` profile support for starter workspace setup and health checks.
- Obsidian default experience pack through `@jarvos/skills`, with doctor and
  install-plan output for optional local commands.
- Release-process hardening: `## [Unreleased]`, release drift detection, release
  readiness checks, and release notes discipline.
- `@jarvos/coding` contracts for tracked issue-to-PR execution: lifecycle
  policy, release/product triage, host adapters, goal-alignment review, and
  review/merge gate equivalents.
- v0.5.0 OpenClaw autonomy install-profile pack, skills reconciliation checks,
  and fresh-host ship-gate smoke coverage.
- Journal safety checks for stale vault paths and single-writer conflicts,
  including Obsidian Daily Notes and Periodic Notes detection.
- Runtime continuity improvements: journal-backed session thread context,
  title-first Paperclip references, and Discord channel-as-context docs.
- Portable manual note maintenance for externally created Markdown notes,
  including dry-run audit, explicit apply mode, GBrain and memory-wiki queue
  cleanup, QMD freshness records, and package/test coverage.

## Fixes

- Resolves the public release drift where `main` had advanced beyond the latest
  GitHub release and root `package.json` still reported `0.3.0`.
- Keeps private Paperclip projects, private GBrain data, machine-local paths,
  and Andrew-specific workspace state out of the public release.
- Makes the full profile clearer: public jarvOS ships the generic contracts,
  adapters, packs, checks, and fixtures; local credentials and private data stay
  with the installer.

## Breaking Changes

- None known.

## Known Limitations

- Distribution is still git/npm-from-repo based; no public npm registry package
  is published yet.
- Full-profile Paperclip and GBrain checks require local credentials and fixture
  configuration. They are documented and smoke-tested, but they are not enabled
  for every fresh clone by default.
- jarvOS remains GBrain-first: it integrates `@jarvos/gbrain` as a resolver and
  runtime context layer, does not implement GBrain itself, and this release is
  not GBrain itself.
- The public repo does not include anyone's private notes, journals, memories,
  goals, beliefs, Paperclip projects, local API keys, or runtime configuration.

## Install / Update

```bash
git clone https://github.com/levineam/jarvOS.git
cd jarvOS
npm test
```

For existing clones:

```bash
cd jarvOS
git pull origin main
npm test
```

After install, try the public CLI:

```bash
node scripts/jarvos.js --help
node scripts/jarvos.js doctor --profile minimal --workspace /path/to/jarvos-workspace
node modules/jarvos-skills/scripts/install-skills.js doctor --json
```
