# jarvOS v0.2.1

## Summary

v0.2.1 is a release-process hardening release. It introduces a clear boundary
between work that is merged on `main` and work that has actually shipped in a
tagged release, so future releases cannot drift into "is this already
released?" ambiguity.

## What's Included

- A `## [Unreleased]` section in `CHANGELOG.md` that serves as the staging area
  for merged-but-unreleased user-facing changes. Cutting a release means moving
  those entries under the new `## v<version>` heading.
- `npm run release:drift` (`scripts/unreleased-drift-check.js`), which detects
  two concrete drift conditions:
  - an untagged release — `package.json` is ahead of the latest git tag with no
    finalized `## v<version>` changelog section to back it; and
  - unlogged work — commits have landed since the latest tag but nothing is
    tracked under `## [Unreleased]`.
  It reports a healthy "ready to tag" state and supports `--json`.
- `docs/release-process.md` now documents the `[Unreleased]` discipline and the
  drift check as part of the standard checklist.

## Fixes

- Closes the release-process gap that allowed work merged to `main` to be
  mistaken for a published release, because nothing distinguished merged work
  from a cut tag.

## Breaking Changes

- None.

## Known Limitations

- The drift check is advisory maintainer tooling; it is intentionally not part
  of `release:check`'s blocking gates, so it never blocks a legitimate release.
- jarvOS remains GBrain-first: it integrates `@jarvos/gbrain` as a resolver and
  runtime context layer, does not implement GBrain itself, and this release is
  not GBrain itself.
- Distribution is still git/npm-from-repo based; no public npm registry package
  is published yet.

## Install / Update

```bash

```

```bash
git pull origin main

```
