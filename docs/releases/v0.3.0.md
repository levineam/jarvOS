# jarvOS v0.3.0

## Summary

v0.3.0 makes jarvOS easier to install, inspect, and trust from the command line.
The release introduces the public `jarvos` command router, a checked-in
`minimal` profile, and `jarvos doctor` checks for the portable starter
workspace.

## What's Included

- `jarvos` is now the public command entrypoint, with `jarvos init`,
  `jarvos doctor`, and `jarvos help`.
- Existing `jarvos-bootstrap` and `jarvos-init` aliases remain available for
  compatibility.
- `jarvos init --profile minimal --yes` routes setup through the public profile
  model while reusing the existing bootstrap installer.
- `jarvos doctor --profile minimal --workspace <path>` checks Node.js,
  required workspace files, `jarvos.config.json`, vault folders, and the public
  `@jarvos/agent-context` package surface.
- `jarvos doctor --json` returns a structured report for agents, scripts, and
  release smoke checks.
- `profiles/minimal.json` describes the first public profile as data, so docs,
  setup, doctor checks, and future profiles can share one contract.
- README usage now shows the CLI happy path for a minimal public workspace.

## Fixes

- New users and agents now have a single public command surface for setup and
  health checks instead of piecing together bootstrap scripts and smoke tests.
- The minimal doctor intentionally excludes local-only Paperclip, private GBrain
  data, and full-profile checks, keeping the public release path portable and
  secret-free.

## Breaking Changes

- None known.

## Known Limitations

- Only the `minimal` public profile is implemented in this release. Codex,
  Claude, Hermes, local OpenClaw, Paperclip, GBrain, and full profiles are still
  future profile lanes.
- `jarvos doctor` validates the public starter workspace and package surfaces;
  it does not inspect private Paperclip state, private vault content, or a live
  local GBrain graph.
- jarvOS remains GBrain-first: it integrates `@jarvos/gbrain` as a resolver and
  runtime context layer and does not implement GBrain itself; this release is
  not GBrain itself.
- Distribution is still git/npm-from-repo based; no public npm registry package
  is published yet.

## Install / Update

```bash
git clone https://github.com/levineam/jarvOS.git
cd jarvOS
npm test
```

For existing clones:

```bash
cd jarvOS
git pull origin main
npm test
```

After install, try the public CLI:

```bash
jarvos init --profile minimal --yes
jarvos doctor --profile minimal --workspace /path/to/jarvos-workspace
```

For local repo testing without a global install:

```bash
node scripts/jarvos.js --help
node scripts/jarvos.js doctor --profile minimal --workspace /path/to/jarvos-workspace
```
