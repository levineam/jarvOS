# jarvOS v0.2.0

## Summary

v0.2.0 combines the Claude Code runtime bootstrap with the first capture
experience milestone: fresh Claude Code sessions get baseline jarvOS context,
and "I have an idea" / "make a note" inputs route into the right portable
markdown surface.

## What's Included

- `runtimes/claude/setup.sh` now materializes `~/.claude/CLAUDE.md` from
  `runtimes/claude/templates/CLAUDE.md.template`.
- CLAUDE template coverage for identity and workspace pointers, governance
  pointers, release-targeting guidance, upstream-evaluation behavior, and common
  jarvOS MCP tooling.
- Setup is idempotent, backs up existing `~/.claude/CLAUDE.md` files before
  overwrite, and preserves local extensions appended below
  `<!-- LOCAL-EXTENSIONS-BELOW -->`.
- New Claude setup controls: `JARVOS_SKIP_CLAUDE_MD=1` and
  `CLAUDE_MD_PATH=<path>`.
- High-confidence idea captures append to the daily journal Ideas section.
- High-confidence note captures create Obsidian-compatible markdown notes and
  add wiki-links back to the daily journal Notes section.
- Medium-confidence capture language routes to the daily journal Flagged section
  for review before any durable note is created.
- `@jarvos/secondbrain` exports the classifier, routing dispatcher, config
  resolver, and skill contracts for `journal-entry`, `note-creation`, and
  `idea-parking`.
- The secondbrain package now resolves vault paths through the repo-owned
  `jarvos.config.json` / `JARVOS_*` resolver.

## Fixes

- The Claude Code runtime setup flow now applies a stable, reusable workspace
  behavior foundation each time sessions start.
- Capture routing now avoids creating durable notes from ambiguous capture
  language before review.

## Breaking Changes

- None known.

## Known Limitations

- CLAUDE.md bootstrapping is currently scoped to Claude Code runtime setup.
- `~/.claude/CLAUDE.md` still supports local manual extension only under the
  `<!-- LOCAL-EXTENSIONS-BELOW -->` marker.
- Medium-confidence captures require human review from the Flagged section.
- GBrain-first recall integration remains a resolver and runtime context layer;
  jarvOS does not implement GBrain itself and this release is not GBrain itself.
- Distribution is still git/npm-from-repo based; no public npm registry package
  is published yet.

## Install / Update

```bash
git clone https://github.com/levineam/jarvOS.git
cd jarvOS
npm test
```

For existing clones:

```bash
cd jarvOS
git pull origin main
npm test
```

For the secondbrain module during local development:

```bash
npm install ./modules/jarvos-secondbrain
```

Configure paths with `JARVOS_VAULT_DIR`, `JARVOS_JOURNAL_DIR`,
`JARVOS_NOTES_DIR`, or `jarvos.config.json`.
