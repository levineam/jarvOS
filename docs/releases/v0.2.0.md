# jarvOS v0.2.0

## Summary

jarvOS now gives every fresh Claude Code session a baseline runtime context via an
idempotent `~/.claude/CLAUDE.md` bootstrap, so runtime behavior and release
targeting rules load alongside hydration context instead of requiring manual setup.

## What's Included

- `runtimes/claude/setup.sh` now materializes `~/.claude/CLAUDE.md` from
  `runtimes/claude/templates/CLAUDE.md.template`.
- CLAUDE template coverage for:
  - Identity and workspace pointers (runtime, sibling runtimes, and core docs)
  - Governance pointers (`AGENTS.md`, `SOUL.md`, `IDENTITY.md`, and runtime CRITICAL-RULES)
  - Release-targeting guidance for `v0.2.0` workflows
  - Upstream-evaluation behavior for candidate-worthy work
  - MCP tooling list for common jarvOS integrations
- Setup is idempotent and preserves local extensions appended below
  `<!-- LOCAL-EXTENSIONS-BELOW -->`.
- Existing `~/.claude/CLAUDE.md` files are backed up before overwrite.
- Two environment controls are added:
  - `JARVOS_SKIP_CLAUDE_MD=1` to skip materialization
  - `CLAUDE_MD_PATH=<path>` to target a custom output path

## Fixes

- The Claude Code runtime setup flow now applies a stable, reusable workspace
  behavior foundation each time sessions start.

## Breaking Changes

- None known.

## Known Limitations

- CLAUDE.md bootstrapping is currently scoped to Claude Code runtime setup.
- `~/.claude/CLAUDE.md` still supports local manual extension only under the
  `<!-- LOCAL-EXTENSIONS-BELOW -->` marker.
- The public repo includes Claude runtime integration code and fixtures, not GBrain
  itself.
- Distribution remains git-based; no npm package is published for this release.
