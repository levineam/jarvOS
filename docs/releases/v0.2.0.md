# jarvOS v0.2.0

## Summary

v0.2.0 makes the first capture experience real: tell an agent "I have an idea"
or "make a note" and jarvOS routes the thought into the right portable markdown
surface.

## What's Included

- High-confidence idea captures append to the daily journal Ideas section.
- High-confidence note captures create Obsidian-compatible markdown notes and
  add wiki-links back to the daily journal Notes section.
- Medium-confidence capture language routes to the daily journal Flagged section
  for review before any durable note is created.
- `@jarvos/secondbrain` exports the classifier, routing dispatcher, config
  resolver, and skill contracts for `journal-entry`, `note-creation`, and
  `idea-parking`.
- The secondbrain package now resolves vault paths through the repo-owned
  `jarvos.config.json` / `JARVOS_*` resolver.
- Claude Code setup now materializes a baseline `CLAUDE.md` while preserving
  local extensions.

## Known Limitations

- Distribution is still git/npm-from-repo based; no public npm registry package
  is published yet.
- GBrain-first recall integration remains a resolver and runtime context layer;
  jarvOS does not implement GBrain itself and this release is not GBrain itself.
- Medium-confidence captures require human review from the Flagged section.

## Install / Update

Install the repo package directly:

```bash
npm install github:levineam/jarvOS
```

For the secondbrain module during local development:

```bash
npm install ./modules/jarvos-secondbrain
```

Configure paths with `JARVOS_VAULT_DIR`, `JARVOS_JOURNAL_DIR`,
`JARVOS_NOTES_DIR`, or `jarvos.config.json`.

## Verification

```bash
npm test
cd modules/jarvos-secondbrain && npm test
npm pack --dry-run
npm run release:check
```
