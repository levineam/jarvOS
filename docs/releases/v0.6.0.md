# jarvOS v0.6.0

## Summary

v0.6.0 is the focused secondbrain release. It makes intentional note and idea
capture a jarvOS-owned path instead of a per-agent patch: supported AI tools can
route captures into canonical Obsidian notes or journals, source-backed
provenance, generated knowledge surfaces, retrieval status, and memory promotion
queues without depending on Andrew-private workspace state.

This release is for people who want the local-first jarvOS secondbrain pattern:
Markdown notes and journals as the human-facing knowledge base, generated
sidecars/wiki/evals as machine-readable support, and durable memory promotion as
a narrower downstream layer.

## What's Included

- Universal intentional capture path for note and idea requests across Claude
  Code, Codex, OpenClaw, Hermes/custom future-agent paths, and unsupported-agent
  fail-closed cases.
- Canonical Obsidian routing: note captures create standalone notes plus exactly
  one journal backlink, and journals stay under `Journal/YYYY-MM-DD.md`.
- Idea routing: lightweight ideas stay in the journal Ideas section, while
  substantive ideas create standalone notes under the same capture contract.
- CaptureEvent v2, provenance metadata, knowledge sidecars, generated LLM-wiki
  inputs, QMD pending state, GBrain queue entries, and secondbrain status
  surfaces for supported capture paths.
- Session-source adapters, capture dispatcher contracts, shared vault
  config/onboarding, package-safe local storage adapters, and a public Paperclip
  client surface.
- Manual-note maintenance documentation and test coverage for integrating
  Markdown notes created outside the canonical writer into the secondbrain stack.
- Retrieval evaluation and status reporting for qmd/wiki/graph-style adapters,
  with Engraph deferred behind a real future adapter/eval gate.
- Public/private boundary documentation for keeping generic jarvOS code public
  while keeping private vault content, Paperclip data, credentials, and local
  paths private.

## Fixes

- Keeps Lobster/OpenClaw as an adapter or enforcement path rather than the owner
  of capture.
- Prevents dangling journal wiki-links when note persistence fails.
- Prevents memory `noteRef` values from pointing at failed note writes.
- Audits manual notes without `updated` frontmatter using filesystem
  modification time.
- Avoids package crashes when optional memory integration is unavailable in a
  standalone secondbrain install.
- Validates `Source Material` as a directory before treating it as the source
  material folder.

## Breaking Changes

- None known for current public users. Before v1.0.0, jarvOS may still make
  pragmatic compatibility changes; this release optimizes for a coherent local
  dogfood and public repo lane.

## Known Limitations

- v0.6.0 does not automatically ingest every AI conversation. The release
  focuses on intentional capture: when a note or idea is created, it enters the
  secondbrain stack correctly.
- Engraph is not production-integrated. It is deferred until a real optional
  adapter measurably improves on qmd plus the generated LLM-wiki.
- Manual-note maintenance has documented dry-run/apply smoke coverage, but no
  hosted Paperclip Routine is enabled by this public repo.
- Distribution is still git/npm-from-repo based; no public npm registry package
  is published yet.
- Public jarvOS does not include anyone's private notes, journals, memories,
  goals, beliefs, Paperclip projects, API keys, or local runtime configuration.

## Install / Update

```bash
git clone https://github.com/levineam/jarvOS.git
cd jarvOS
npm test
```

For existing clones:

```bash
cd jarvOS
git pull origin main
npm test
```

Useful secondbrain checks:

```bash
cd modules/jarvos-secondbrain
npm test
npm pack --dry-run
```
