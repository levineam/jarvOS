## Summary

v0.7.0 is the post-v0.6.3 control-plane release: an authenticated
control-plane application service, a protected-resource mutation policy
layer, and public human/agent parity for that service.

## User-facing parity impact

Humans and supported agent harnesses now use one authenticated,
capability-filtered control-plane path instead of adapter-owned lifecycle
shortcuts. Operators request/approve/inspect through `jarvos-manager`;
agents use the `jarvos_control_plane` MCP tool. Both bind credentials
server-side against a host-provided application service module—public
packages ship the adapters and contracts, not private host state.

## Included

- **SUP-3478 / #103** — Core authenticated control-plane application-service
  boundary for adapter request envelopes. Approvals bind to action key,
  capability, expiry, and fence; fail closed on replay/staleness. List/inspect
  projections filter by principal sensitivity.
- **SUP-3493 / #107** — Protected-resource mutation policy layer for
  `@jarvos/control-plane`: resource identity, owning adapter, allowed named
  operations, structured allow/deny/fail-closed decisions with evidence
  digests, Daily Journal identity matcher pattern.
- **SUP-3493 / #108** — Opt-in journal section-transform seam
  (`jarvos-secondbrain-journal`) for surgically cleaning recovered Notes
  scaffold through maintenance APIs.
- **SUP-3466 / #106** — Public control-plane parity (merged as
  `03ea5df597`). Human CLI (`jarvos-manager`) and agent MCP
  (`jarvos_control_plane`) share the capability-filtered application-service
  path, with host service-module binding, server-side credential resolution,
  doctor/setup readiness checks, and module/runtime docs.
