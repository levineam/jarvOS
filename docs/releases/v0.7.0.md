# jarvOS v0.7.0 (release candidate — not yet cut)

Paperclip release lane: SUP-3497. This document tracks release-candidate
scope while the lane is active; it is not a finalized release and must not be
treated as published until the checklist in `docs/release-process.md` is
green and Andrew has approved publication.

## Summary

v0.7.0 is the post-v0.6.3 control-plane release: an authenticated
control-plane application service, a protected-resource mutation policy
layer, public human/agent parity for that service, and a portable coding
manager compatibility layer for supported agent hosts.

## User-facing parity impact

Humans and supported agent harnesses now use one authenticated,
capability-filtered control-plane path instead of adapter-owned lifecycle
shortcuts. Operators request/approve/inspect through `jarvos-manager`;
agents use the `jarvos_control_plane` MCP tool. Both bind credentials
server-side against a host-provided application service module—public
packages ship the adapters and contracts, not private host state.

## What's Included

- **SUP-3478 / #103** — Core authenticated control-plane application-service
  boundary for adapter request envelopes. Approvals bind to action key,
  capability, expiry, and fence; fail closed on replay/staleness. List/inspect
  projections filter by principal sensitivity.
- **SUP-3493 / #107** — Protected-resource mutation policy layer for
  `@jarvos/control-plane`: resource identity, owning adapter, allowed named
  operations, structured allow/deny/fail-closed decisions with evidence
  digests, Daily Journal identity matcher pattern.
- **SUP-3493 / #108** — Opt-in journal section-transform seam
  (`jarvos-secondbrain-journal`) for surgically cleaning recovered Notes
  scaffold through maintenance APIs.
- **SUP-3466 / #106** — Public control-plane parity (merged as
  `03ea5df597`). Human CLI (`jarvos-manager`) and agent MCP
  (`jarvos_control_plane`) share the capability-filtered application-service
  path, with host service-module binding, server-side credential resolution,
  doctor/setup readiness checks, and module/runtime docs.
- **SUP-3470 / #112** — Public `@jarvos/coding` control-plane compatibility
  for Claude Code, Codex, and OpenClaw host adapters. The portable manager
  resumes from durable checkpoints, fails closed on deferred or incomplete
  submission/closeout evidence, and reasserts target fences at pull-request,
  post-merge, and issue-close mutation boundaries.
- **SUP-3496 / release-intake follow-up** — Release-trust hardening treats
  placeholder-only `[Unreleased]` sections as empty and removes the historical
  v0.3/SUP-1957 release parent from unconfigured coding triage. Missing release
  configuration now stays visible and fail-closed.

## Known Limitations

- Public CLI/MCP control-plane surfaces require an installed host that
  configures `JARVOS_CONTROL_PLANE_SERVICE_MODULE` and binds credentials;
  public/minimal installs do not ship private host state.
- The coding manager ships public contracts and adapters, not private
  repository credentials, Paperclip wiring, or host-specific session stores.
- This candidate is not published: no version bump, tag, or GitHub Release
  until release-process checklist is green and Andrew approves publication.
