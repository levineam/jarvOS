# jarvOS v0.6.2

## Summary

v0.6.2 is a patch candidate for secondbrain capture determinism across AI
coding tools. It makes the current target explicit: OpenClaw, Codex, Claude
Code, Hermes, and future coding adapters should all use jarvOS-owned capture
semantics for intentional notes and ideas.

This is post-v0.6.1 hardening. It does not broaden jarvOS into automatic
ingestion of every AI conversation, and it does not make general chat
applications active secondbrain determinism targets.

## What's Included

- Adds `CODING_TOOL_SOURCE_TOOLS` as the active source list for deterministic
  secondbrain capture: `openclaw`, `codex`, `claude-code`, and `hermes`.
- Removes general chat app names from the active CaptureEvent source enum; future
  non-coding assistants can start as `custom:<slug>` experiments.
- Adds a first-class Hermes session-source adapter alongside OpenClaw, Codex,
  and Claude Code.
- Adds `docs/contracts/CODING_TOOL_DETERMINISM.md`, which records the
  OpenClaw-derived behavior as a jarvOS-owned public contract.
- Updates Codex, Claude Code, Hermes, and OpenClaw runtime docs/templates so
  intentional captures call `jarvos-capture` or the compatibility contract.
- Adds a drift smoke that fails if docs/templates/source constants drift away
  from the coding-tool capture contract.

## Known Limitations

- v0.6.2 does not automatically ingest all AI conversations.
- v0.6.2 does not integrate general chat apps as active first-class capture
  targets.
- The compatibility note/journal contract remains available for older
  personality writers while runtime adapters move toward `jarvos-capture`.
- The release is not published until the release checklist is green on a clean
  tree and Andrew approves public publication.
