# TOOLS.md — Tool & CLI Reference

## What belongs here

- **Tool CLI patterns** (commands, flags, usage examples)
- **Tool-specific config** (env vars, paths, accounts)
- **Lint/compliance script references** (how to run checks)

## What does NOT belong here

- Policy, process, or governance rules → AGENTS.md or agents/*.md
- Model routing or spawn policies → agents/system.md
- Planning or project management rules → agents/projects.md
- Content or writing rules → agents/content.md

**Before adding a section, ask: "Is this a tool or CLI reference?"** If no, route it to the correct file. See AGENTS.md § Where Rules Live.
